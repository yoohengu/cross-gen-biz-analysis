# 데이터 상세사항
- customers_hm
    - `customer_id`: 고객 id
    - `FN`: 패션 뉴스 구독 여부
    - `Active`: 커뮤니케이션 가능 여부
    - `club_member_status`: 회원 상태(신규, 활성, 탈퇴)
    - `fashion_news_frequency`: 패션 뉴스 알림 주기
    - `age`: 나이

# 과정
문제 정의
→ 데이터 이해/수집
→ 전처리
→ EDA
→ 분석
→ 인사이트
→ 시각화/전달
→ 결론

1️⃣ 문제 정의 (Problem Definition)
무엇을 알고 싶은지 / 해결하고 싶은지 명확히 하기
분석 목적을 한 문장으로 정리
✔️ 예시
“매출 감소의 주요 원인을 파악한다”
“고객 이탈에 영향을 주는 요인을 분석한다”
🔥 이 단계가 흐리면 뒤에서 분석이 산으로 감

2️⃣ 데이터 이해 및 수집 (Data Understanding & Collection)
어떤 데이터가 있는지 파악
데이터 구조 확인
컬럼 의미
데이터 타입
행 수, 결측치 여부
데이터 출처 정리 (CSV, DB, API 등)
✔️ 이 단계에서는 분석하지 말고 ‘구경’만 한다는 느낌이 좋아

3️⃣ 데이터 전처리 (Data Cleaning & Preprocessing)
결측치 처리
이상치 확인 및 처리
데이터 타입 변환
필요한 컬럼 생성 (파생 변수)
분석에 불필요한 데이터 제거
💡 실제 프로젝트에서 가장 시간 많이 쓰는 단계

4️⃣ 탐색적 데이터 분석 (EDA)
데이터의 분포와 패턴 파악
주요 지표 요약 (평균, 중앙값, 비율 등)
그룹별 비교 (카테고리별, 기간별 등)
시각화 적극 활용
🔎 “이 데이터는 대체 어떤 성격이지?”를 파악하는 단계

5️⃣ 분석 / 모델링 (Analysis or Modeling)
문제 정의에 맞는 분석 수행
비교 분석
상관관계 분석
간단한 통계 분석
(필요하다면) 머신러닝 모델
가설 검증 or 패턴 도출
✔️ 기술보다 ‘왜 이 분석을 하는지’가 더 중요

6️⃣ 결과 해석 및 인사이트 도출 (Insight)
분석 결과를 말로 설명
“그래서 뭐가 중요한데?”에 답하기
수치 → 의미 → 시사점 흐름으로 정리
❌ “A의 평균은 3.2입니다”
⭕ “A가 높은 그룹일수록 구매 전환율이 높았습니다”

7️⃣ 시각화 및 커뮤니케이션 (Visualization & Communication)
핵심 결과를 그래프로 정리
한 장에 하나의 메시지
비전공자도 이해할 수 있게 설명
🎯 분석은 혼자 하지만, 결과는 남에게 보여주는 것

8️⃣ 결론 및 한계점 정리
프로젝트 요약
얻은 인사이트 정리
데이터/분석의 한계
다음에 더 해볼 수 있는 분석 제안
