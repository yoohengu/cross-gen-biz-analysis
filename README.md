# 🛒 Cross-Gen-Biz-Analysis : H&M 데이터 기반 연령대별 트렌드 분석
> **온/오프라인 채널과 연령대별 상호작용 분석을 통한 비즈니스 전략 도출**

---

### 1. 📋 프로젝트 개요
본 프로젝트는 **H&M의 고객 구매 데이터**를 활용하여, 연령대별 온/오프라인 소비 패턴과 트렌드를 심층 분석합니다. 단순한 현상 파악을 넘어, 판매 채널과 연령대가 실제 구매액에 미치는 상호작용 효과를 검증하고 이를 바탕으로 한 타겟 프로모션 전략 제안을 목표로 합니다.

### 2. 🎯 주요 목적 및 목표
* **세그먼트 분석**: 연령대별 핵심 키워드를 도출하고 고객 그룹핑을 통한 맞춤형 전략 수립
* **채널 영향력 파악**: 온라인과 오프라인 채널 선호도가 구매액에 미치는 영향 분석
* **상호작용 효과 검증**: 연령대와 판매 채널 간의 상관관계를 파악하여 최적의 마케팅 믹스 제안

### 3. 👥 팀 구성 및 역할
| 성함 | 역할 | 담당 데이터 및 업무 |
| :--- | :--- | :--- |
| **유현규 (팀장)** | 프로젝트 총괄 | `customers` 테이블 전처리 & 분석 및 통합 |
| **조은솔 (부팀장)** | 데이터 고도화 | `articles` 테이블 전처리 & 분석 상품별 트렌드 분석 |
| **이찬희** | 라이브 발표 | `transactions` 테이블 전처리 & 분석 및 성과 스피칭 |

### 4. 🚀 분석 프로세스
1. **데이터 탐색(EDA)**: 각 담당 테이블별 기초 데이터 확인 및 전처리
2. **키워드 도출**: 연령대별 구매 패턴에서 핵심 비즈니스 키워드 추출
3. **인사이트 통합**: 개별 분석 결과를 결합하여 온/오프라인 통합 전략 도출

### 📁 5. 폴더 구조 및 작업 규칙

팀원들 간의 원활한 협업을 위해 아래 규칙을 준수해 주세요!

* **`notebooks/`**: **메인 분석 공간**
    * 개인별 분석 코드(`.ipynb`)를 저장합니다.
    * 파일명 앞에 **본인 이름**을 붙여서 구분해 주세요. (예: `yoonju_transactions_eda.ipynb`)
    * 혹시나 불편할 경우 꼭 개별 폴더를 생성하여 관리해주세요.
* **`data/`**: **개인 로컬 데이터 보관소**
    * 100MB가 넘는 대용량 원본 데이터는 깃허브에 올라가지 않습니다.
    * 팀원 각자 본인의 컴퓨터 내 이 폴더에 직접 데이터 파일을 넣어주세요.

### 🚨 필수 주의사항
1.  **작업 시작 전 `git pull`**: 다른 팀원이 올린 내용을 먼저 내 컴퓨터로 가져와야 충돌이 발생하지 않습니다.
2.  **환경 동기화**: 팀원이 환경을 변경했다면 터미널에서 `uv sync`를 입력해 분석 환경을 맞춰주세요.
